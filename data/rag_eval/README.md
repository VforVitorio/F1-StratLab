# data/rag_eval

RAG agent evaluation set and report for chapter 5 of the TFG thesis. The ground-truth queries are manually verified against the FIA Sporting Regulations PDFs; the report compares three retriever configurations head to head.

## Files

- `queries_v1.json` — 15 Spanish queries with ground truth verified as literal substrings of the corresponding PDF article. Schema: `id`, `year`, `query`, `category`, `ground_truth.{article, section, expected_keywords, rationale}`. Editable only if you re-verify against the PDFs in `data/rag/documents/`.
- `results_v1.md` — auto-generated by `notebooks/agents/N30B_rag_benchmark.ipynb` (Restart Kernel -> Run All). Contains the comparative summary table and discussion section ready to copy into chapter 5.

## How to regenerate the report

Open `notebooks/agents/N30B_rag_benchmark.ipynb`, Restart Kernel -> Run All. The final cell writes `results_v1.md` over the existing file.

## Adding a new query

1. Open the relevant FIA Sporting Regulations PDF in `data/rag/documents/`.
2. Locate the article the query maps to and copy its section identifier.
3. Confirm that every entry in `expected_keywords` is a literal substring of that article (case-insensitive). If not, rephrase the keywords until they are.
4. Append the new object to `queries_v1.json` with a fresh `id` and the same schema as the existing entries.
5. Re-run `notebooks/agents/N30B_rag_benchmark.ipynb` to refresh `results_v1.md`.
